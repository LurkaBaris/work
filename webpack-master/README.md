# webpack

*Сборка проекта*

Разработка команда : npm run dev

*Требования к проекту*
- требования к написанию стилей https://nicothin.pro/idiomatic-pre-CSS/
- весь js пишется на классах под es6
- Проект разбит в соотвествии с bem блоками используется синтаксис two-dash
	- элемент блока  __
	- модификатор --

*Структура*

- source/assets - набор изображений/шрифтов/svg - которые нужно собирать  - в изображениях только дизайнерские фитчи(которые не будут зависить от контента)
- source/base - стили / скрипты - которые не зависят от блоков. Какие собственные плагины. (Обратить внимание на _functions.scss) Пример в блоке core-container
- source/blocks - блоки которые будут использоваться на сайте. - название блока должно ложиться на файловую структуру к примеру возьмем блок catalog-filter - соотвественно в структуре будет
blocks -> catalog -> filter -> catalog-filter.pug(а так же рядом лежать catalog-filter.scss и catalog-filter.js)
- source/core -> по аналогии с blocks  - тут находятся блоки низшего уровня - или ядровые блоки. Расположение по аналогии с blocks
- source/data -> данные для построения pug страниц и содержимого страниц. Разбивать json на смысловые блоки для его переиспользования пример находится в проекте.
- source/entity - стили и скрипты для каждой страницы - должны содержать минимальное кол-во необходимых блоков для рендера страницы
- source/entity/common/common.js - общие скрипты для шаблона
- source/entity/common/common.scss - общие стили для шаблона
- source/layouts - шаблоны страниц
- source/pages - сами pug страницы
- source/static - статичные файлы которые необходимы для сборки - копируются в корень сборки.
- source/static/ajax - json для асинхронных ответов формы апи и так далее
- source/static/assets/images - папка для контентных изображений
